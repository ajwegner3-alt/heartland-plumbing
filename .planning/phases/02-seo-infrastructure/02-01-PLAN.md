Create `src/lib/schema/` directory with 5 generator files. Each exports a single function returning `WithContext<T>` from schema-dts.

**local-business.ts** — `generateLocalBusinessSchema(): WithContext<LocalBusiness>`
- Import BUSINESS from '@/lib/data/business'
- Use '@type': 'Plumber' (more specific subtype of LocalBusiness)
- Include ALL fields: name, url, telephone, address (PostalAddress), geo (GeoCoordinates), openingHoursSpecification (weekday + saturday as two specs), aggregateRating (AggregateRating inline), areaServed (array of strings), priceRange, sameAs (socialLinks array)
- If schema-dts types are strict about certain fields, cast with `as unknown as Type` per research recommendation

**service.ts** — `generateServiceSchema(service: ServiceData): WithContext<Service>`
- Import ServiceData type from '@/lib/data/services'
- Import BUSINESS from '@/lib/data/business'
- Include: name, description (shortDescription), url (constructed from BASE + slug), provider (LocalBusiness with name/phone/url), areaServed (City: Omaha, containedInPlace: State: Nebraska), offers with priceSpecification using service.pricingRange

**faq-page.ts** — `generateFAQPageSchema(faqs: Array<{q: string; a: string}>): WithContext<FAQPage>`
- Map each faq to Question with acceptedAnswer Answer
- Accept the same {q, a} shape that services.ts already uses

**breadcrumb.ts** — `generateBreadcrumbSchema(items: BreadcrumbItem[]): WithContext<BreadcrumbList>`
- Export interface BreadcrumbItem { name: string; url: string }
- Map items to ListItem with position (1-indexed)
- Use BUSINESS.url as BASE for constructing full URLs, OR accept full URLs from caller

**aggregate-rating.ts** — `generateAggregateRatingSchema(): WithContext<AggregateRating>`
- Import BUSINESS for rating values
- Standalone schema (not embedded in LocalBusiness) for pages that show reviews without the full LocalBusiness context
- Note: This is separate from the inline aggregateRating within LocalBusiness schema

**JsonLd.tsx** — A simple Server Component (NO 'use client'):
```tsx
export function JsonLd({ data }: { data: object }) {
  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(data) }}
    />
  )
}
```
Do NOT use next/Script — it causes RSC payload duplication per research findings.

IMPORTANT: If schema-dts types cause friction on nested objects (e.g., `offers` on Service), use targeted type assertions rather than abandoning the library. The JSON-LD output is what matters for Google — TypeScript strictness is a bonus, not a blocker.
