    **Step 2: Build `src/components/ContactForm.tsx`** — a `'use client'` component:
    - State: `useState<'idle' | 'submitting' | 'success' | 'error'>('idle')` and `errorMsg` string
    - Fields: name (text, required), phone (tel, required), service (select dropdown with options: Drain Cleaning, Water Heaters, Sewer Line Repair, Emergency Plumbing, Other), zip (text, optional), message (textarea, optional)
    - Honeypot: hidden input named `_gotcha` with `tabIndex={-1}`, `autoComplete="off"`, `aria-hidden="true"`, `style={{ position: 'absolute', left: '-9999px', opacity: 0 }}`
    - Client-side validation before fetch: name must not be empty, phone must have 10+ digits after stripping non-digits. Show inline error messages below each field.
    - On submit: `e.preventDefault()`, set status to 'submitting', POST to `/api/contact` with JSON body via fetch, handle response
    - Success state: Replace entire form with a green success panel — checkmark icon, "Message Sent!", "We'll call you within 1 hour", and a "Send Another Message" button that resets to idle
    - Error state: Show error message above submit button, keep form visible so user can retry
    - Submit button: disabled during 'submitting' state, shows "Sending..." text
    - Form field styling: Match hero form exactly — `border border-border rounded-sm px-3.5 py-2.5 text-sm text-text-primary font-body placeholder:text-text-muted/60 focus:outline-none focus:border-primary transition-colors`
    - Phone field should be first after name (most important field for contractors)
    - Import service slugs from services data if possible, or hardcode the 4 services + "Other"

    **Step 3: Build `src/app/api/contact/route.ts`** — POST route handler:
    - Initialize Resend client at module scope: `const resend = new Resend(process.env.RESEND_API_KEY)`
    - Guard: if `!process.env.RESEND_API_KEY`, return 500 with `{ error: 'Email not configured.' }`
    - Parse body with `request.json()`
    - Honeypot check: if `body._gotcha` has value, return `{ success: true }` (200) silently
    - Validate required fields: name and phone must be non-empty after trim
    - Validate phone: `phone.replace(/\D/g, '').length >= 10` — return 400 if too short
    - Call `resend.emails.send()` with:
      - `from`: `process.env.RESEND_FROM_EMAIL || 'Heartland Plumbing <onboarding@resend.dev>'`
      - `to`: `[BUSINESS.email]` (imported from business.ts)
      - `subject`: `New Lead: ${name} — ${service || 'General Inquiry'}`
      - `html`: `buildLeadEmail({ name, phone, service, zip, message })` (imported from lead-notification.ts)
    - Check `error` from Resend response, log and return 500 if present
    - Return `{ success: true }` on success
    - Wrap everything in try/catch for unexpected errors

    **Step 4: Build `src/app/contact/page.tsx`** — Server Component page shell:
    - Metadata via `generatePageMetadata()`: title "Contact Us | Heartland Plumbing Co." (under 60 chars), description 150-155 chars with CTA, path "/contact"
    - BreadcrumbList schema: Home > Contact
    - Page layout:
      - Banner section: dark teal bg, H1 "Contact Heartland Plumbing", breadcrumb, subtitle about getting in touch
      - Two-column layout on desktop (lg:grid-cols-2), single column on mobile:
        - Left column: ContactForm component
        - Right column: Contact info card with phone (tel: link), address, hours (from BUSINESS), and a "For emergencies, call us directly" note
      - Trust signals below form: license, insurance, rating badges (small inline version)
    - Import and render ContactForm component
